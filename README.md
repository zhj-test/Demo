# SnailApp

这个平台实现python编写的测试用例的执行/管理功能。
注意，平台没有实现编写用例的功能，要把自己写的项目测试用例，放到tests目录下.

环境准备：

1. python3环境
2. 项目目录下建venv虚拟环境
3. ./venv/bin/pip install -r requirements
4. sh ./init_migration.sh      #创建迁移仓库
5. sh ./create_migration.sh    #创建迁移脚本
6. sh ./do_migration.sh        #迁移
7. sh ./gunicorn.sh            #启动   （sh ./start-debug.sh 启动调试模式）

使用说明：

测试代码约定:

    1. 在tests目录下，新建文件夹，存放一个项目的测试代码。
    2. 将python编写的测试代码文件复制到新建的项目目录下， 最好按功能模块划分文件， 一个模块的测试用例写在一个py文件中。
    3. 模块测试用例文件，以类的形式编写，一个py文件中可有一个或多个类
    4. 类中的方法为测试用例方法， 以assert方式进行断言判断用例是否通过
    5. 如用例执行时，没有异常抛出，判定为用例pass.
    6. 如抛出异常为AssertException, 判定为Failed.
    7. 如抛出其它异常，判定为Error.

平台操作
1. 在平台上新建项目，输入项目的目录等信息.
2. 在项目信息页， 新建模块， 指定模块的文件名(不带.py后缀名)，类名等信息，对应测试代码的py文件，文件中的类
3. 模块下新建用例，指定用例对应的方法名和参数名(多个参数以|分隔)。
4. 在用例下新建测试数据，给出用例参数对应的传入数据(数据间以|分隔)，注意，传入的参数都为string类型
5. 项目-模块-用例-测试数据，从左到右都是一对多关系。
6. 可在项目下新建测试集，选取项目下的用例。
7. 可执行测试集，执行测试集下的每一条用例,并生成测试报告。一个测试集可多次执行，生成多个测试报告，可以理解为测试轮次的概念。
8. 项目-测试集-测试轮次，从左到右都是一对多的关系。
